## Problem Encountered

**Error on GitHub Actions:**
```
clang-14: error: no such file or directory: 'jnius/jnius.c'
error: command '/path/to/clang' failed with exit code 1
```

**What This Meant:**
- Local build passed successfully (49 MB APK created)
- GitHub Actions build proceeded further but failed at PyJNIus compilation
- The C source file `jnius.c` was missing
- This file MUST be generated by Cython from `jnius.pyx` before setup.py can compile

---

## Root Cause Analysis

### Why It Failed
1. PyJNIus is a Cython extension (Python code that wraps Java via C)
2. PyJNIus source includes `.pyx` files (Cython source)
3. PyJNIus source DOES NOT include `.c` files (compiled C source)
4. The `.c` files must be generated during build via Cython compilation
5. Previous recipe was skipping the Cython compilation step
6. When setup.py tried to compile the C extension, the `.c` files didn't exist

### Why Local Build Succeeded
- Local environment may have had cached Cython-generated files
- Or the Cython compilation happened through a different code path
- GitHub Actions uses clean ephemeral containers (no cache)

## Solution Implemented

### The Fix
**File**: `android_recipes/pyjnius/__init__.py`

**Changes Made**:

```python
# BEFORE: Missing Cython step

    # Removes pyproject.toml
    # Calls setup.py install ← FAILS: jnius.c not found!

# AFTER: Proper sequence

    # 1. Recipe.build_arch() - NDK setup
    # 2. self.build_cython_components() ← CRITICAL NEW STEP
    # 3. Remove pyproject.toml
    # 4. setup.py install ← WORKS: jnius.c now exists!
```

### Exact Changes

1. **Added import**:

2. **Added Recipe initialization**:
   ```python

   ```

3. **Added Cython compilation** (THE CRITICAL FIX):
   ```python

   self.build_cython_components(arch)
   ```

4. **Maintained setup.py force logic**:
   - Remove pyproject.toml (after Cython, not before)
   - Call setup.py install

## Technical Reference

### Why Cython Is Needed
- PyJNIus provides Python access to Java objects
- Java-Python bridge requires C code
- Cython simplifies writing C extensions
- PyJNIus source is in Cython (.pyx)
- Build process: `.pyx` → (Cython) → `.c` → (clang) → `.so`

### Why Order Matters
1. **NDK must be initialized first** - Provides compiler toolchain
2. **Cython must run second** - Generates C files
3. **Build config must be removed third** - Forces setup.py
4. **setup.py install runs last** - Has everything it needs
