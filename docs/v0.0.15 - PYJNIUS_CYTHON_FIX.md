# GitHub Actions Build Error Fix - PyJNIus Cython Compilation

## Issue Identified

**Error**: `clang-14: error: no such file or directory: 'jnius/jnius.c'`

**Root Cause**: PyJNIus is a Cython-based extension that needs to compile `.pyx` files to `.c` files before `setup.py install` can build the C extension. The custom recipe was skipping the Cython compilation step.

---

## Solution Implemented

### Updated PyJNIus Recipe
**File**: `android_recipes/pyjnius/__init__.py`

**Key Changes**:
1. Added `Recipe.build_arch(self, arch)` - Runs base NDK/recipe setup
2. Added `self.build_cython_components(arch)` - **CRITICAL**: Compiles `.pyx` → `.c` files
3. Remove modern build config (pyproject.toml, setup.cfg)
4. Call `setup.py install` only after Cython compilation

### Build Order (Correct Sequence)
```
1. Base Recipe setup (NDK configuration)
   ↓
2. Cython compilation (.pyx → .c)  ← FIX: This was missing
   ↓
3. Remove pyproject.toml (force setup.py)
   ↓
4. setup.py install (build C extension using setup.py)
   ↓
5. PyJNIus ready for inclusion in APK
```

---

## Technical Details

### What Cython Does
- Converts PyJNIus Cython source files (`.pyx`) to C source files (`.c`)
- Generates the necessary binding code for calling Java from Python
- Creates compiled Python extension modules

### Why It Was Missing
- The original `build_arch()` override was skipping the parent's build flow
- The parent `CythonRecipe` class normally handles this via `build_cython_components()`
- Our override wasn't calling the essential Cython step

### Why It Works Now
- `Recipe.build_arch()` ensures NDK/toolchain is properly set up
- `build_cython_components()` generates `.c` files from `.pyx` files
- `setup.py install` can now find the required C source files
- Everything happens in the correct order and context
