# Android Build Failure Fix - Summary

## Problem
The Android build was failing with:
```
ERROR Backend 'setuptools.build_meta:__legacy__' is not available.
Cannot import 'setuptools.build_meta'
```

This occurred during the PyJNIus build when using the modern PEP 517 build backend with an isolated build environment.

## Root Cause Analysis
1. PyJNIus source includes `pyproject.toml` declaring `setuptools.build_meta` as build backend
2. python-for-android's CythonRecipe detects pyproject.toml and uses `python -m build` (PEP 517)
3. This creates an isolated environment where setuptools fails to properly install/import
4. The build fails before it even gets to the compilation stage

## Solution - Three-Part Fix

### 1. Custom PyJNIus Recipe Override
**File:** `android_recipes/pyjnius/__init__.py`

**Changes:**
- Override `build_arch()` method to completely bypass modern build backend
- Remove `pyproject.toml`, `setup.cfg`, and `MANIFEST.in` files
- Use traditional `setup.py install` directly, matching the PythonRecipe pattern
- Properly configure environment and install paths

**Why it works:**
- By removing the pyproject.toml before calling build, we eliminate the PEP 517 detection
- Calling `setup.py install` directly avoids the isolated environment mechanism
- Uses the same build methodology as successful Python packages in p4a

### 2. Enable setup.py Support in buildozer.spec
**File:** `buildozer.spec`

**Change:**
```diff
- p4a.setup_py = false
+ p4a.setup_py = true
```

**Effect:**
- Changes p4a flag from `--ignore-setup-py` to `--use-setup-py`
- Tells python-for-android to prioritize setup.py over modern build systems
- Works in conjunction with our recipe override

### 3. Pre-install Build Dependencies in GitHub Actions
**File:** `.github/workflows/build.yml`

**Changes:**
- Added pre-installation of setuptools, wheel, and cython
- Placed before buildozer commands
- Provides fallback mechanism in case isolated environment is still triggered

**Why it helps:**
- Ensures necessary build tools are available systemwide
- If isolated environment is created, it has a better chance of succeeding
- Defensive measure for robustness

## Files Modified
1. `android_recipes/pyjnius/__init__.py` - Custom recipe override
2. `buildozer.spec` - Enable setup.py support
3. `.github/workflows/build.yml` - Pre-install build dependencies
