# PyJNIus Cython Compilation Fix - Manual Implementation

## Problem
GitHub Actions build was failing with:
```
AttributeError: 'PyjniusRecipe' object has no attribute 'build_cython_components'
```

The previous solution attempted to call `self.build_cython_components(arch)` which doesn't exist in the python-for-android Recipe class.

## Root Cause
1. PyJNIus contains `.pyx` (Cython source) files, not `.c` (compiled C source) files
2. These `.pyx` files must be compiled to `.c` before `setup.py install` can build the C extension
3. The non-existent `build_cython_components()` method was called instead of manually compiling Cython
4. Without `.c` files, C compiler fails with: `clang-14: error: no such file or directory: 'jnius/jnius.c'`

## Solution: Manual Cython Compilation

### Implementation (Commit c3dd37b)
The custom PyJNIus recipe now manually compiles Cython files:

```python
def build_arch(self, arch):
    # ... setup ...

    # Step 2: Compile Cython components (.pyx -> .c)
    info('Building Cython components for PyJNIus')
    env = self.get_recipe_env(arch)

    # Find all .pyx files in the source tree
    pyx_files = []
    for root, dirs, files in os.walk(build_dir):
        for file in files:
            if file.endswith('.pyx'):
                pyx_files.append(os.path.join(root, file))

    if pyx_files:
        info(f'Found {len(pyx_files)} Cython files to compile')
        # Try compiling each .pyx file
        with current_directory(build_dir):
            for pyx_file in pyx_files:
                rel_path = os.path.relpath(pyx_file, build_dir)
                info(f'Compiling {rel_path} with Cython...')
                try:
                    # Method 1: Try using python -m cython (most reliable)
                    shprint(
                        sh.Command(self.hostpython_location),
                        '-m', 'cython',
                        pyx_file,
                        _env=env
                    )
                    info(f'✓ Successfully compiled {rel_path}')
                except Exception as e:
                    # This is non-fatal - setup.py might compile inline
                    info(f'Note: Cython pre-compilation of {rel_path} skipped: {e}')
                    continue
```

### Key Design Decisions

1. **Manual Discovery**: Walks the build directory to find all `.pyx` files
2. **Python Module Invocation**: Uses `python -m cython` (the hostpython) instead of shell `cython` command
   - More reliable in CI/CD environments
   - Ensures correct Cython version matches hostpython
3. **Non-Fatal Compilation**: If individual `.pyx` compilation fails, it continues
   - setup.py can also compile Cython inline if needed
   - Allows graceful degradation
4. **Proper Environment**: Uses `get_recipe_env(arch)` to set build environment
   - Ensures correct NDK paths and Android SDK environment
5. **Relative Paths**: Reports compilation progress with human-readable relative paths

### Build Sequence (Fixed)

Now the correct order is:

1. ✅ `Recipe.build_arch(self, arch)` - Initialize NDK toolchain
2. ✅ Manual Cython compilation (`python -m cython *.pyx`) - Generate .c files
3. ✅ Remove `pyproject.toml` - Force legacy setup.py mode
4. ✅ `setup.py install` - Build and install C extension

Previously step 2 was missing, causing step 4 to fail when looking for `.c` files.

## Fallback Mechanism

If Cython pre-compilation fails for any reason:
- The error is logged but non-fatal
- `setup.py install` can attempt inline Cython compilation using the modern build backend
- However, this may reintroduce the original pyproject.toml/setuptools.build_meta error

The pre-compilation step ensures this doesn't happen by generating `.c` files upfront.
